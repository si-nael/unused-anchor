# v0.5.5 Endogenous Causal Branching

Date: 2026-07-17

Status: completed bounded endogenous-causal-branching release on 2026-07-17

## Connected Seam

`v0.5.4` deliberately stopped when simultaneously enabled internal laws had order-dependent effects. The causal runtime returned `underdetermined` because declaration order, host iteration order, randomness, or an authored branch menu would all install an external selector.

`v0.5.5` closes one narrower continuation seam: a Bubble may now realize plural autonomous worlds directly from its own simultaneously enabled law structure.

This is not deliberative agency. No entity chooses an option, no preference ranks internal branches, and no protagonist or story is introduced. It is an exact world-formation rule for cases where the world law itself admits more than one mutually non-commuting actualization.

## Maximal-Commuting-Frontier Derivation

Programs opt into the new contract with:

```json
{
  "execution": {
    "internalConflictMode": "maximal-commuting-branches"
  }
}
```

The default remains `underdetermined`, preserving every earlier `v0.5.2` through `v0.5.4` program.

For one enabled-law set:

1. each enabled law becomes a vertex;
2. two vertices are connected exactly when their complete effect sets commute;
3. the runtime enumerates every maximal clique of that compatibility graph;
4. each maximal clique becomes one realized causal frontier;
5. every simultaneously enabled law excluded from that frontier is recorded as nonrealized in that branch and is not silently executed later;
6. every branch continues causal closure under the same program until closed, contradicted, or honestly unresolved.

The derivation is exhaustive and deterministic but not selective. A law compatible with every alternative belongs to every maximal frontier. Identical `set` projections and zero additive effects commute; a `set` mixed with a meaningful additive change, or unequal `set` projections on the same field, does not.

The runtime records a `CausalInternalBranchStep` containing:

- all enabled law ids;
- the realized maximal frontier;
- the mutually excluded nonrealized laws;
- `derivation: maximal-commuting-frontier`;
- `hostSelection: false`.

This branch lineage is carried by autonomous, selected, inspected, stored, and replayed continuations.

## Exact Resource Boundary

`maxInternalBranches` bounds generated branch states, and `maxInternalFrontiers` bounds realized causal frontiers across the closure tree. If the runtime cannot preserve every maximal frontier under either bound, it returns `underdetermined`.

It never returns the first enumerated branch as a convenient result. `resourceUse` separately records branch count, branch limit, frontier count, and whether internal branching was exhaustive.

## World Will Over Plural Worlds

Endogenous worlds are not candidate actions for World Will to collapse.

Each autonomous branch receives its own:

- objective baseline;
- anchor-identity evaluation;
- eligible intervention set;
- exact intervention search.

If one intervention group produces further endogenous branches, those outcomes remain one set-valued consequence. World Will compares that intervention by its minimum exact net improvement across every endogenous outcome:

`guaranteed improvement = min(outcome score - branch baseline score - intervention cost)`.

An intervention group is improving only when this guaranteed value is positive. If selected, every one of its endogenous outcomes survives. World Will may choose among intervention groups according to the existing deterministic or plural decision contract, but it may not choose an internally generated branch outcome.

## What v0.5.5 Does Not Claim

This release does not establish:

- deliberation or an internal chooser among simultaneous alternatives;
- agency, life, consciousness, protagonist identity, relationship, or story coherence;
- dynamic law/schema generation, populations, reproduction, or evolution;
- concrete `spawn` or `collapse` execution in the causal-world kernel;
- a greatest or infinite viability kernel;
- general coinduction or open-ended branching;
- global multi-world positive/negative-sea transport;
- cross-world anchor transfer or a complete identity calculus;
- universal physical time;
- whole Bubble-universe completion;
- recursive module/code/self construction.

The next release must be chosen from the remaining connected Bubble-universe gaps. Phase 2 observer and comparison work remains deferred.
