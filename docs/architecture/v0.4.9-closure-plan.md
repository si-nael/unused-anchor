# v0.4.9 Closure Plan

Date: 2026-07-15

Status: completed on 2026-07-15

## Decision

`v0.4.9` is required before `v0.5`.

It is a convergence release, not a broad feature release. Every completion gate below is now satisfied and the package version is `0.4.9`.

## Why It Is Required

At the `v0.4.8` boundary, effect provenance linked authored effects to concrete runtime targets. The runtime could inspect negative-sea pressure, positive-sea support, anchor state, observation, collapse, commit, and effect causality, but it could not yet state what class of source best explained an observed event without a human reconstructing those surfaces manually.

That gap is Q-014 and one of the original purposes of the sea-anchor model. Carrying it unresolved into `v0.5` would make the ontology descriptive where it was intended to discriminate.

## Bounded Semantic Scope

Add one runtime source-attribution assessment with this vocabulary:

- `internal-world-event`
- `negative-sea-pressure`
- `anchor-drift`
- `positive-sea-shift`
- `unresolved-source`

The assessment must:

1. derive only from authored sources, materialized evidence, causal links, and runtime ontology present in the same run
2. distinguish a candidate from a resolved attribution
3. preserve multiple candidates and choose `unresolved-source` when available evidence cannot discriminate them
4. avoid treating a weak anchor as proof that drift occurred
5. avoid treating every authored perturbation as foreign residue
6. expose its basis through materialization, inspection, stored replay, and report filtering

## Research Continuity Scope

The release also closes the record defects found before implementation:

- correct the D-082 date boundary
- record closure-first release discipline
- add explicit repository memory
- log the README encoding boundary failure
- reconcile stale question statuses
- maintain the idea/question implementation map
- add automated checks for core record structure and date ordering

## Non-Goals

Do not add any of the following in `v0.4.9`:

- direct authored sea-law syntax
- continuous or multi-step sea dynamics
- a full anchor identity calculus
- deterministic replay re-execution or stochastic replay
- coupled multi-region collapse
- probabilistic-fractal semantics
- first-class observer agents
- new effect primitives

## Completion Gates

`v0.4.9` is complete only when:

- source attribution has a typed runtime representation and evidence record
- fixtures cover every supported resolved class plus at least one genuinely unresolved case
- every attribution basis resolves to current-run evidence, ontology, or authored IR
- inspect and replay preserve the assessment without changing its verdict or basis
- record-structure verification and the full `npm run verify` path pass
- README, reference, semantic specification, question/hypothesis logs, work log, decision log, project memory, and package version agree on the completed boundary
- generated run artifacts are refreshed and inspected for semantic plausibility

## Completion Evidence

- typed `event-source-attribution` evidence is implemented for collapse, commit, descendant, and observation subjects
- direct and contextual candidates preserve typed basis references
- focused fixtures resolve all four supported direct classes and preserve genuine ambiguity as `unresolved-source`
- weak anchor state and merely declared perturbation do not independently become executed causes
- inspection and replay expose `sourceAttributions` plus status and classification filters
- `attribution-crossroads.bubble` is in mandatory `verify:attribution` materialize, inspect, record, and replay paths
- record preservation checks keep the original idea clusters, open questions, deferred research, and the `v0.5` successor rule visible
- the complete repository verification path passes at the release boundary

## After v0.4.9

The next version is `v0.5`. Open it around one measurable research objective. The strongest current candidate is a scored event-source or same-world judgment benchmark comparing Bubble's explicit ontology and evidence path against a realistic host-language implementation. That would begin testing whether Bubble earns its existence instead of merely widening its syntax.

This succession does not close the larger user ideas behind sea dynamics, anchor identity, observer research, unbounded worldhood, or probabilistic-fractal semantics. Their original records and explicit reopen conditions remain authoritative.
